## 二、受眾分析

| 客群 | 規模 | 痛點 | 付費意願 |
|------|------|------|---------|
| **森林組合 (森林合作社)** | ~620 組織 | 調查人力不足、精度低、成本高 | 高 |
| **自治體林務課** | ~1,700+ 市町村 | 森林簿更新困難、缺乏數位工具 | 高 (B2G) |
| **J-Credit 計畫申請者** | 急增中 (大規模計畫增多) | 需要精確的胸高直徑 (DBH) 資料做碳量計算 | 極高 |
| **林業公司/素材生產業者** | ~數千社 | 伐採計畫需要資源量調查 | 中-高 |
| **大學/研究機構** | ~50+ | 生態調查需要效率工具 | 中 |
| **海外林業 (東南亞、歐洲)** | 巨大 | 熱帶林/人工林管理 | 中 (價格敏感) |
| **環境 NGO** | ~數百 | 森林監測、違法伐採偵測 | 低 (需贊助模式) |

### 2.1 關鍵用例：J-Credit 碳信用

```
J-Credit 森林經營計畫流程：

1. 森林調查 (DBH + 樹高 + 樹種 + 位置)  ← 你的 App 直接切入
2. 碳吸收量計算 (材積 → CO2 換算)        ← 你的 SaaS 可以做
3. 計畫登錄 + 第三方審查
4. クレジット認證 + 交易

傳統：2 人組隊 + 巻尺 + 記錄表 → 1 天 ~50 棵
你的方案：1 人 + iPhone + 360 相機 → 1 天 200-500 棵
```

## 四、差異化分析

### 4.1 vs ForestScanner (森林總研)

這是你最直接的「先行者」— 同名、同概念、已有學術驗證。

| 維度 | 森林總研 ForestScanner | 你的 ForestScanner |
|------|----------------------|-------------------|
| DBH 測量 | ✅ LiDAR (R²=0.963) | ✅ LiDAR (相同原理) |
| 樹種辨識 | ❌ | ✅ YOLO + CoreML |
| 360° 全景輸入 | ❌ | ✅ 360° → 6 分鏡 → SfM |
| 碳量/材積計算 | ❌ | ✅ DBH + 樹高 → 材積 → CO2 |
| GIS 輸出 | ❌ (僅 CSV) | ✅ GeoJSON → SaaS 地圖 |
| 雲端 SaaS | ❌ | ✅ 既有 GIS 平台整合 |
| J-Credit 報表 | ❌ | ✅ 直接產出申請用報表 |
| 商業模式 | 免費研究工具 | SaaS 訂閱 + B2G 標案 |

**結論：森林總研版是「量測工具」，你做的是「端到端林業數位化平台」。**

## 六、市場前景

### 6.1 順風因素

| 因素 | 影響 |
|------|------|
| **J-Credit 需求爆發** | 自然系碳信用市場 2030 年 ¥320 億，目前供不應求 |
| **林業イノベーションハブ** | 林野庁 2025 年全國 10 地域實證 |

| **日立等大廠驗證市場** | 大廠進場 = 市場被驗證，但他們做 B2B 高端 |
| **航空 LiDAR 開放資料** | 政府公開資料 → 可結合地面掃描做比對 |
| **GX リーグ** | 440+ 企業參加，碳交易制度化 |

### 6.2 逆風因素

| 因素 | 對策 |
|------|------|
| **與森林總研 ForestScanner 同名** | 品牌差異化 (Edge-to-Cloud 平台 vs 量測工具) |
| **iPhone LiDAR 精度不如專業設備** | 已有學術驗證「精度足夠」(RMSE 2.3cm) |
| **林業從業者不熟手機** | 極簡 UI + 引導式操作 + 現場培訓 |
| **偏鄉網路不穩** | 離線作業 → 回家同步 (你的「現場採集、回家處理」) |
| **日本 IT 採購保守** | B2G 標案先取得自治體信任 |
